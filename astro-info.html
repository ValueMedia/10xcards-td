Astro

Framework webowy nowej generacji - szybkosc przez prostote

Historia

Astro zostal stworzony przez Freda Schotta i Nate'a Moore'a - tworcow Snowpack i Skypack. Pierwsza publiczna beta zostala ogloszona 8 czerwca 2021 roku pod haslem "Ship Less JavaScript".

  • Czerwiec 2021 - Pierwsza publiczna beta. Koncepcja "Islands Architecture"
  • Sierpien 2022 - Astro 1.0 - stabilna wersja produkcyjna
  • Styczen 2023 - Astro 2.0 - Content Collections, tryb Hybrid
  • Sierpien 2023 - Astro 3.0 - View Transitions, optymalizacja obrazow
  • Marzec 2024 - Astro 4.0 - Dev Toolbar, i18n routing
  • Grudzien 2024 - Astro 5.0 - Content Layer, Server Islands, Vite 6
  • 2025 - Astro 6.x - dalszy rozwoj wydajnosci i DX

Astro jest projektem open source (licencja MIT), wspieranym przez The Astro Technology Company i spolecznosc 900+ kontrybutorów.

Dla kogo jest Astro?

Idealne zastosowania:

  • Tworcy stron content-driven - blogi, strony marketingowe, dokumentacja, portfolio
  • Zespoly e-commerce - szybkie strony produktowe z dynamicznymi elementami
  • Deweloperzy SEO-first - strony wymagajace doskonalej wydajnosci
  • Zespoly wielotechnologiczne - projekty laczace React, Vue, Svelte
  • Agencje webowe - szybkie dostarczanie wydajnych stron
  • Tworcy dokumentacji - Starlight jako standard branzy

Astro NIE jest najlepsze dla:

  • Aplikacji typu SPA z intensywna interaktywnoscia (dashboardy, edytory)
  • Aplikacji real-time wymagajacych ciaglego WebSocket
  • Gier przegladarkowych lub aplikacji 3D

Glowne cechy

Islands Architecture

Cala strona to statyczny HTML. JavaScript laduje sie tylko dla interaktywnych "wysp" - niezaleznie od siebie.

Zero JS domyslnie

Brak JavaScript w przegladarce, chyba ze jawnie go zazadasz. Strony laduja sie blyskawicznie.

Bring Your Own Framework

Uzywaj React, Vue, Svelte, Preact, Solid lub Lit w jednym projekcie. Kazdy komponent w innej technologii.

Content Layer

Type-safe zarzadzanie trescia z dowolnego zrodla: Markdown, CMS, API, bazy danych. Jedno API.

Server Islands

Statyczny HTML + dynamiczne komponenty serwerowe. Personalizacja bez rezygnacji z cache.

View Transitions

Natywne animacje przejsc miedzy stronami bez dodatkowego JavaScript.

Wydajnosc budowania

Do 5x szybszy build Markdown, 25-50% mniej pamieci. Vite 6 pod spodem.

astro:env

Type-safe zmienne srodowiskowe z walidacja, rozdzieleniem client/server i ochrona sekretow.

Ekosystem

500+ integracji. Adaptery: Cloudflare, Vercel, Netlify, Node.js, Deno.

Przyklady uzycia

1. Blog z Content Collections

---
// src/pages/blog/[slug].astro
import { getCollection } from 'astro:content';
import Layout from '../../layouts/Layout.astro';

export async function getStaticPaths() {
  const posts = await getCollection('blog');
  return posts.map(post => ({
    params: { slug: post.id },
    props: { post },
  }));
}

const { post } = Astro.props;
const { Content } = await post.render();
---
<Layout title={post.data.title}>
  <article>
    <h1>{post.data.title}</h1>
    <Content />
  </article>
</Layout>

2. Server Islands (e-commerce)

---
// Statyczne + dynamiczne na jednej stronie
import Product from '../../components/Product.astro';
import Cart from '../../components/Cart.astro';
---
<Layout>
  <!-- Statyczne, cachowane -->
  <Product id={Astro.params.id} />

  <!-- Dynamiczne Server Island -->
  <Cart server:defer>
    <p slot="fallback">Ladowanie koszyka...</p>
  </Cart>
</Layout>
