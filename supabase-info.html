Supabase

Open Source Firebase Alternative

Build in a weekend. Scale to millions.

Historia

Supabase zostal zalozony w styczniu 2020 przez Paula Copplestone'a (CEO) i Ant Wilsona (CTO). Projekt narodzil sie z potrzeby otwartej alternatywy dla Firebase, zbudowanej na sprawdzonym PostgreSQL.

  • Styczen 2020 - Zalozenie firmy. Wczesne prototypy z PostgREST i Realtime
  • Kwiecien 2020 - Akceptacja do Y Combinator (batch S20)
  • Wrzesien 2020 - Publiczna beta: Database + Auth + Realtime
  • Wrzesien 2021 - Series A ($30M). Wprowadzenie Storage
  • Maj 2022 - Series B ($80M). Launch Edge Functions (Deno)
  • Kwiecien 2023 - Studio v2, Local Dev z CLI, Branching
  • Kwiecien 2024 - Supabase Vector (pgvector + AI toolkit)
  • 2025 - SOC2 Type 2, 16+ regionow, 1M+ baz danych, S3-compatible Storage

Supabase jest open source (Apache 2.0). Kazdy komponent mozna self-hostowac niezaleznie.

Dla kogo jest Supabase?

Idealne zastosowania:

  • Startupy i indie hackerzy - szybki start, darmowy tier, kompletny backend w godzine
  • Full-stack deweloperzy - backend bez pisania serwera, auto-generowane API
  • Zespoly migrujace z Firebase - podobne DX, ale z relacyjna baza SQL
  • Aplikacje real-time - natywne WebSocket (presence, broadcast, DB changes)
  • Projekty AI/ML - pgvector, embeddings, wyszukiwanie semantyczne
  • Enterprise - SOC2, RLS, self-hosting, audit logs
  • Deweloperzy React/Next.js/Astro - dedykowane SDK i integracje SSR

Supabase NIE jest najlepsze dla:

  • Aplikacji wymagajacych wlasnosciowego modelu danych (graf, document-first)
  • Ekstremalnie wysokiego write throughput (miliony zapisow/sekunde)
  • Prostych statycznych stron bez backendu

Auto-generowane API

REST (PostgREST), GraphQL (pg_graphql) i Realtime (WebSocket) - automatycznie z kazdej tabeli, bez pisania kodu.

Row Level Security

Autoryzacja na poziomie wierszy w SQL. Polityki bezpieczenstwa w jednym miejscu, nie rozrzucone po kodzie.

Supabase Studio

Graficzny interfejs: Table Editor, SQL Editor, zarzadzanie uzytkownikami, logami, metrykami.

CLI + Local Dev

Pelny stos lokalnie w Docker. Migracje, branching, type generation, seeding.

Pelna portowalnosc

Standardowy PostgreSQL - eksport danych i przeniesienie gdziekolwiek. Zero vendor lock-in.

SOC2 + Enterprise

SOC2 Type 2, szyfrowanie, network restrictions, IP allowlisting, audit logs.

Przyklady uzycia

1. Polaczenie z baza danych

// Inicjalizacja klienta
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  'https://your-project.supabase.co',
  'your-anon-key'
);

// Pobranie danych z relacjami
const { data, error } = await supabase
  .from('products')
  .select('id, name, price, category(name)')
  .eq('active', true)
  .order('created_at', { ascending: false })
  .limit(10);

2. Autoryzacja (Auth)

// Rejestracja
const { data, error } = await supabase.auth.signUp({
  email: 'user@example.com',
  password: 'secure-password',
});

// Social login
const { data, error } = await supabase.auth.signInWithOAuth({
  provider: 'google',
  options: { redirectTo: 'https://myapp.com/callback' }
});

3. Row Level Security (SQL)

-- Uzytkownik widzi tylko swoje dane
CREATE POLICY "Users see own data" ON profiles
  FOR SELECT
  USING (auth.uid() = user_id);

-- Uzytkownik edytuje tylko swoje posty
CREATE POLICY "Users edit own posts" ON posts
  FOR UPDATE
  USING (auth.uid() = author_id)
  WITH CHECK (auth.uid() = author_id);

4. Realtime (nasluchiwanie zmian)

// Subskrypcja INSERT na tabeli posts
const channel = supabase
  .channel('posts-changes')
  .on('postgres_changes',
    { event: 'INSERT', schema: 'public', table: 'posts' },
    (payload) => {
      console.log('Nowy post:', payload.new);
    }
  )
  .subscribe();

// Presence - kto jest online
const presenceChannel = supabase.channel('room-1');
presenceChannel
  .on('presence', { event: 'sync' }, () => {
    const state = presenceChannel.presenceState();
    console.log('Online:', Object.keys(state));
  })
  .subscribe();

5. Integracja z Astro (projekt 10xcards)

// src/lib/supabase.ts
import { createClient } from '@supabase/supabase-js';

export const supabase = createClient(
  import.meta.env.SUPABASE_URL,
  import.meta.env.SUPABASE_KEY
);

// Uzycie w Astro API route
export const GET: APIRoute = async ({ request }) => {
  const { data, error } = await supabase
    .from('flashcards')
    .select('*')
    .order('created_at', { ascending: false });

  return new Response(JSON.stringify(data));
};

6. Wyszukiwanie wektorowe (AI)

-- Tabela z embeddingami
CREATE TABLE documents (
  id BIGSERIAL PRIMARY KEY,
  content TEXT,
  embedding VECTOR(1536)
);

-- Wyszukiwanie semantyczne
CREATE FUNCTION match_documents(
  query_embedding VECTOR(1536),
  match_count INT
) RETURNS TABLE (id BIGINT, content TEXT, similarity FLOAT)
LANGUAGE plpgsql AS $$
BEGIN
  RETURN QUERY
  SELECT d.id, d.content,
    1 - (d.embedding <=> query_embedding) AS similarity
  FROM documents d
  ORDER BY d.embedding <=> query_embedding
  LIMIT match_count;
END;
$$;
